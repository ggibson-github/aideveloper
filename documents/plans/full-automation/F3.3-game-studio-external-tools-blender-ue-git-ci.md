## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When a game-studio pack is active, pursuit loads declared external tools—Blender, Unreal Engine, Perforce or Git, and CI build farms—from pack YAML plus role bindings in [F6.3](F6.3-role-tool-permissions-mcp-cli-allowlist.md) before spawning tool-operator workers.
2. Compose-first resolves pack playbooks and facts INDEX entries ([E1.4](E1.4-catalog-facts-index.md)) for install paths and credential locations before any worker improvises MCP or CLI commands.
3. Tool-operator and verifier workers execute bounded exports, engine tests, and CI invocations under the role allowlist; they attach evidence per [F6.4](F6.4-role-evidence-requirements-per-output-type.md) but never mutate pursuit state in journal or state.json.
4. Preflight blocks spawn when a required tool is missing from the host, an MCP server is down, or credentials are absent—raising H2 with facts INDEX pointers rather than skipping the task.
5. If allowlist drift, credential expiry, or tool failure leaves evidence incomplete, pursuit stays fail-closed at H2 until operators reconcile tool bindings at H1 and workers re-run with passing verifier output.

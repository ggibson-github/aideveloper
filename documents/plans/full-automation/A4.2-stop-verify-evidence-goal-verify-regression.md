**Parent:** [A4-index](A4-index.md) · **Branch A** · **Vision §3** · **Release:** v2.15

## Reader narrative
<!-- prose-source: agent plane-a 2026-06-28 -->

Verification stops protect quality when evidence or goal-level proof fails. Task evidence missing, `verify-router` failure, goal_verify regression, or conformance script exit non-zero all produce a hard stop with structured H2—not a silent retry loop.

Fail-closed behavior prevents false “achieved” states that would reach H3 without proof. The conductor records which command failed and which evidence paths were absent so the operator or a resumed pursuit knows exactly what to fix. This stop class connects Plane A to Plane G; it is distinct from resource exhaustion (A4.3) and integrity corruption (A4.4).

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When preflight finds missing task evidence (`evidence_required` true but paths absent), [G1.1](G1.1-task-verify-router-verifier.md) or verify-router exit non-zero, `goal_verify_command` failure, or [G2.4](G2.4-goal-verify-regression-every-implement-batch.md) regression, check-pipeline-blocked emits stop class `verify:*` and exits the [A2.6](A2.6-loop-until-blocked-budget-achieved-h3-reject.md) loop without advancing `next_action`.
2. Journal-keeper dual-writes the failed command, missing evidence paths, and `last_verify=failed` to journal and state.json so resumed pursuit or the operator knows exactly what to fix—fail-closed, no silent retry loop.
3. Task-level verify failures block implement task completion; goal-level failures block [A2.5](A2.5-goal-verify-pass-transition-h3-pending.md) H3 pending transition per [G2.3](G2.3-goal-verify-blocks-h3-until-pass.md), preventing false achieved states that would reach H3 without proof.
4. This stop class connects Plane A pursuit to Plane G verification—it is distinct from resource exhaustion ([A4.3](A4.3-stop-resource-max-steps-max-cost-lease-expired.md)) and state corruption ([A4.4](A4.4-stop-integrity-validate-workflow-state-corrupt.md)).
5. If verify stops fire but evidence later passes on re-run, pursuit resumes only after preflight confirms all required proof paths exist—never auto-clearing H2 without successful verifier output.

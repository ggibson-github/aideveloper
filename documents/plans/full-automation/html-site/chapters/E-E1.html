E1 · Knowledge & composition plane

Catalog (single discovery surface)

E1: Catalog (single discovery surface). Open each capability below for diagrams, verification, and contracts.

The scripts catalog is a generated manifest under scripts/ that lists every S0-capable entry point—route-tier, validate-workflow, verify-router, hierarchy automation—so conductors and compose queries discover deterministic behavior without grep-ing the tree.

Regeneration runs via S0 when scripts change; the manifest feeds E2.2 list-components and the umbrella E1.7. Stale manifest rows trigger E5.2 graph updates. See Vision §7 — Branch E — Knowledge & composition plane.

The pipelines manifest under docs/manifest/pipelines/ declares pipeline_id values—greenfield, iterative_feature, program—with phase order, default skills, and verify suites. Instantiation sets pipeline_id in state.json; route-tier selects the active pipeline for next_action.

This catalog binds Plane E discovery to Plane C execution (C1.4). Pack-defined pipelines differ per company template without forking conductor code. Changes mark dependent task cards stale via E5.1 and may require reconcile-stale (E5.3).

Each template-pack publishes a README cataloging pack fragments: roles, pipeline overrides, verify suites, and governance defaults. README files under template-packs/*/ are discoverable catalog nodes for compose and pack instantiation.

Pack READMEs complement the generated platform umbrella (E1.7) with consumer-specific bindings (B5.2). Promotion from L0 reasoning should target pack fragments listed here. Staleness extends to pack nodes when vision or parent pack schema changes (E5.2).

docs/platform/CATALOG.md is the generated umbrella index merging scripts, playbooks, skills, facts, pipelines, and pack README entries into one discovery surface. S0 regeneration keeps the umbrella current when any child catalog changes.

Operators and conductors start compose queries here before deep tree walks. The umbrella feeds E2.2 and satisfies the single discovery surface goal of E1. Cross-links to child INDEX files must stay bidirectional. Platform maturity claims in Plane D depend on umbrella completeness and staleness graph coverage (E5.1).

F6 · Organization plane

Role → agent mapping

F6: Role → agent mapping. Open each capability below for diagrams, verification, and contracts.

Role mapping keeps one genius-tier conductor while state.company.active_role switches context—pipeline_id, spawn policy, and HITL strictness follow the active role from pack bindings (B5.1). Workers swap personas per spawn; conductor merges summaries and dual-writes journal/state.

Context switch is not a new chat session—state.json carries active_role across turns. Switching mid-lease requires lane release (F2.3). Mis-mapping (platform permissions on product role) is conformance failure. Same conductor identity enables centralized S4 H2 packaging (B5.4).

Per-role allowed_reads scopes workers to role playbooks, lane task cards, facts entries, and design slices declared in pack YAML—not the full vision monolith or unrelated department lanes. Librarian returns paths within this cap (max five) before implement spawn (B2.2).

Scope bleed (QA worker reading finance playbooks) violates pack intent and risks cross-department edits. allowed_reads updates on active_role rotation (F6.1). Lane.json work orders may further narrow reads for parallel slots. Forbidden paths are explicit deny list alongside allowed_reads.

Tool permissions per role encode MCP server allowlists, CLI patterns, and write scopes—programmer may run build CLI; designer may run Blender MCP; governance may read-only audit tools. Bindings live in roles/*.yaml and mirror operator policy overrides (B5.2).

Violations block spawn or fail verifier with permission error—never silent downgrade to broader tools. External game tools (F3.3) and data deploy CLIs declare here. Pack authors test permission matrix with conformance scripts before publish.

Evidence requirements per role output type define what verify must prove for each deliverable—code diff + pytest for programmer tasks, export checksum + render for TA tasks, schema migration log for DBA tasks. Requirements extend generic evidence_required in state with pack-specific output_type rows.

Task cards reference output_type; verify-router and tool-operator write matching evidence paths (G-plane task evidence). goal_verify (F1.8) aggregates role evidence at scope complete. Missing output_type evidence blocks task completion even when generic last_verify passed—fail closed per role contract.

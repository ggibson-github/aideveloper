E2 · Knowledge & composition plane

Composition protocol (mandatory before S1+)

E2: Composition protocol (mandatory before S1+). The capabilities below follow a sequential flow—read them in order.

Before any S1+ work, compose must resolve which capability the turn needs: routing, implement, verify, design artifact, platform promotion. Resolution names the capability id and target catalog slice—not a vague "figure it out" prompt.

Librarian and conductor share this step; output drives E2.2 query and E2.3 ranking. Unresolved capability blocks spawn until catalog search completes. This implements mandatory compose-before-invent from B4.3 at the Plane E protocol layer.

Catalog query runs through S0 list-components.py (or successor) against the umbrella and child INDEX manifests—returning candidate paths, maturity tier, and component type. Queries are deterministic; conductors use output rather than re-deriving hits in chat.

Input is the resolved capability from E2.1; output feeds E2.3 ranking and task card Components (E2.4). Empty results trigger E2.5 L0 proceed plus promotion enqueue—not silent invention.

Compose ranking orders catalog hits: script (S0) first, then playbook, then skill, then facts. v2.24+ extends this via E6.5: hard_transistor > script > soft_transistor > playbook > skill > facts.

Ranking applies after E2.2 list-components returns hits. Ties break toward higher maturity and pack-authored fragments. Divergence from ranked choice requires B4.4 log entry. Ranking aligns with capability_class S0–S4 in Plane B.

The compose plan lands on the task card Components section: explicit catalog refs (script path, playbook id, skill name) bound before implement spawn. Components used is auditable evidence that compose-first ran—not optional prose.

Conductor or task-breakdown writes Components after E2.3 ranking. Workers inherit listed refs in allowed_reads (E4.3). Missing components when catalog had matches is a conformance failure; missing catalog match pairs with E2.5 promotion enqueue.

When compose finds no suitable catalog component, pursuit proceeds at L0 (ephemeral reasoning) for the immediate turn—but must enqueue a promotion_queue item so Plane D can distill the pattern into script, playbook, or pack fragment later.

This closes the reuse loop: product is not blocked, platform debt is recorded. Pair with B4.4 divergence logging and B4.2 K-step drain. Repeated L0 for the same capability signals catalog gap—prioritize promotion over repeated improvisation.

game studio external tools Blender UE Git CI

Game studio packs declare external tools: Blender, Unreal Engine, Perforce or Git, CI build farms—MCP servers, CLI allowlists, and tool-operator task card patterns per role (F6.3). Tool-operator and verifier workers run bounded exports and engine tests; they do not edit pursuit state.

Missing tool install or credentials is H2—not skip. Facts INDEX holds download URLs and credential locations (E1.4). Evidence for tool runs follows F6.4 output types (render checksum, export log, CI artifact path). Compose-first searches pack playbooks before improvising tool commands.

How it works

  1. Step 1When a game-studio pack is active, pursuit loads declared external tools—Blender, Unreal Engine, Perforce or Git, and CI build farms—from pack YAML plus role bindings in F6.3 before spawning tool-operator workers.
  2. Step 2Compose-first resolves pack playbooks and facts INDEX entries (E1.4) for install paths and credential locations before any worker improvises MCP or CLI commands.
  3. Step 3Tool-operator and verifier workers execute bounded exports, engine tests, and CI invocations under the role allowlist; they attach evidence. but never mutate pursuit state in journal or state.json.
  4. Step 4Preflight blocks spawn when a required tool is missing from the host, an MCP server is down, or credentials are absent—raising H2 with facts INDEX pointers rather than skipping the task.
  5. Step 5If allowlist drift, credential expiry, or tool failure leaves evidence incomplete, pursuit stays fail-closed at H2 until operators reconcile tool bindings at H1 and workers re-run with passing verifier output.

D4.2 · Platform evolution plane

platform, work, script, and extraction

platform work script extraction

Script extraction promotes L1 playbook steps—or verified manual commands—into S0 Python under scripts/ with tests. Extraction is platform work: changes tooling for all future goals, not the current consumer feature branch unless explicitly shared.

Extracted scripts must be idempotent, CLI-invokable, and referenced from playbook diff. Wire verify-router when script replaces task-card Test command. Backwards-compatible extensions use D5.2; behavior breaks require D5.3 fork with new catalog id. B1.1 mandates conductor runs script before model re-derivation.

How it works

  1. Step 1On platform drain for script-extraction items, an economy worker promotes stable L1 playbook steps or verified manual commands into idempotent S0 Python under scripts/ with tests.
  2. Step 2The worker wires verify-router when the extracted script replaces a task-card Test command so future implement turns invoke S0 instead of ad-hoc shell.
  3. Step 3Playbook diff references the new script path; the conductor enforces B1.1—run the script before the model re-derives the same behavior.
  4. Step 4Backwards-compatible behavior changes follow D5.2; breaking changes require D5.3 fork with new catalog id and provenance.
  5. Step 5If extraction lacks tests, a CLI entry point, or playbook linkage, the queue item stays open and pursuit blocks false L2 promotion at H2.

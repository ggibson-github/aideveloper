A3.2 · Pursuit & control plane

goal autopilot, until, goal verify, or, hard, and block

goal_autopilot until goal_verify or hard block

Goal autopilot is the north-star run mode: after H1 approves a plan, pursuit continues automatically until goal_verify passes, a hard block fires, or a budget from A1.4 is exhausted. Unlike session autopilot, there is no arbitrary per-chat step ceiling—the stop taxonomy in A4 decides termination.

Each iteration still runs exactly one pipeline phase (A2.2) after S0 preflight (A2.1). Scope completion triggers goal-level verification (A2.4), not another implement task. Operators enable this via autopilot.active in state.json or the /autopilot skill once the goal model from A1 is populated.

How it works

  1. Step 1After H1 approves the plan and the A1 goal model is populated, the operator enables goal_autopilot via autopilot.active with pursuit.mode set to goal_autopilot—lifting the per-session ceiling from A3.1 so pursuit continues across wakes until termination.
  2. Step 2Each wake runs A2.1 preflight, exactly one A2.2 pipeline phase, and A2.3 dual-write—matching A2.7 one-phase-per-turn semantics across unlimited iterations without waiting for manual Continue.
  3. Step 3When machine-checkable scope completion fires, A2.4 runs goal_verify_command via G2.1 instead of scheduling another implement-feature turn.
  4. Step 4Termination follows the A4 stop taxonomy—hard blocks become H2, A1.4 budget caps dual-write structured stop reasons, and goal_verify success transitions to A2.5 H3 pending.
  5. Step 5If goal_verify fails, preflight cannot classify a stop reason, or required evidence goes stale mid-run, pursuit halts at H2 and goal.state stays pursuing until scope and proof reconcile.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • goal_verify passes but H3 rejected — goal.state returns to pursuing with rejection notes.
  • Edge case A3.2 variant 4: verify state dual-write before continuing pursuit.

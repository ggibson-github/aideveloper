B5.3 · Cognition & routing plane

handoff, manifest, artifact, and graph

handoff manifest artifact graph

Handoff manifest and artifact graph declare cross-workstream dependencies: which design nodes must exist before implement lanes start, which integration tests gate program completion. Program orchestration reads the manifest before parallel spawn (orchestrate-program skill).

Stale nodes in the graph trigger reconcile-artifact-graph before workers proceed. Handoff is the human-approved integration contract for multi-stream company goals.

How it works

  1. Step 1After H1 approves a program milestone, program-scoper and integration-manifest-keeper produce the handoff manifest and artifact_graph listing which design nodes must exist before each implement lane may start and which integration tests gate program completion.
  2. Step 2Before orchestrate-program spawns parallel lanes, the conductor reads the manifest and refuses spawn for any lane whose upstream artifact_graph nodes are absent, unverified, or still marked stale.
  3. Step 3When upstream design or implementation changes a dependency, reconcile-artifact-graph marks affected nodes stale and pauses only the blocked lanes until reconciliation completes and state.json reflects cleared stale flags.
  4. Step 4Each workstream records satisfied artifact_graph nodes and integration evidence in state under its lane lease (C4.1) so company-level goal_verify can roll up completion against manifest gates.
  5. Step 5If the manifest human gate was never cleared, dependency edges cycle, or a lane starts without satisfied upstream nodes, pursuit stops at H2—never running uncoordinated parallel implement across streams.

D4.5 · Platform evolution plane

platform, work, catalog, and regeneration

platform work catalog regeneration

Catalog and INDEX regeneration is platform work after new playbooks, scripts, skills, or pack fragments land: update docs/playbooks/INDEX.md, manifest catalog nodes, and operator discovery rows so compose-first search finds fresh artifacts (B4.3).

Regeneration is S0 where scripts exist; otherwise economy worker with allowed_reads on new artifact paths only. Regeneration without artifact creation is invalid—INDEX must point to real files. Librarian and implement workers depend on fresh INDEX rows; skipping regeneration breaks B4.3 and fails D6.1.

How it works

  1. Step 1After new playbooks, scripts, skills, or pack fragments land, platform work runs S0 catalog regeneration when scripts exist; otherwise an economy worker with allowed_reads limited to new artifact paths only.
  2. Step 2The worker updates docs/playbooks/INDEX.md, manifest catalog nodes, and operator discovery rows so B4.3 compose-first search finds fresh artifacts.
  3. Step 3Every INDEX row must resolve to a real file on disk—regeneration without artifact creation is invalid and fails D6.1.
  4. Step 4On verify pass, the conductor dequeues the promotion item; librarian and implement workers on subsequent product turns see updated catalog paths.
  5. Step 5If INDEX rows are stale, broken, or point at missing files, pursuit blocks downstream compose-first routing at H2 until regeneration completes.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • Platform queue depth 0 but product blocked on missing playbook — D3.3 priority cut skips platform drain.
  • Edge case D4.5 variant 4: verify state dual-write before continuing pursuit.

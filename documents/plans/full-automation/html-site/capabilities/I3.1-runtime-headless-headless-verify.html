runtime headless headless-verify

scripts/headless-verify.py runs verify-router and validate-workflow without an IDE session—used locally and as the pattern for CI verify steps. Headless mode reads state.json and task cards from disk; it does not spawn conductor LLM turns unless explicitly configured.

Output writes to evidence/ with the same immutability contract as IDE verifier (H4). Failures set last_verify failed in state for conductor resume on next IDE session.

How it works

  1. Step 1Scheduled CI or an operator invokes scripts/headless-verify.py, which reads state.json, task cards, and configured evidence paths from disk without spawning a conductor LLM turn unless explicitly configured.
  2. Step 2The script runs B1.1 verify-router and validate-workflow for the active pursuit step, executing the same Test commands task cards declare for IDE verifier workers.
  3. Step 3Passing runs append immutable logs under evidence/ per the H4 contract so headless output aggregates into goal_verify the same way IDE-captured evidence does.
  4. Step 4On failure, headless-verify sets last_verify to failed in state.json and records log paths so the conductor can resume on the next IDE continue without assuming prior checks still hold.
  5. Step 5If state.json is corrupt, required task cards are missing, or evidence paths are stale, headless mode fails closed and surfaces structured H2 rather than marking verification passed.

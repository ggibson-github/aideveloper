validate-workflow-dag.py verifies DAG acyclicity, typed edge compatibility, gate edge completeness, and transistor preconditions satisfiable from goal inputs—before implement starts.

See Vision §7 — Branch E — Knowledge & composition plane and SEC-18-transistor-model-a-to-z.

  nodeVerify -->|fail| h2[H2 or retry gate]

How it works

  1. Step 1Run at workflow-compose phase exit and on any DAG edit before resume.
  2. Step 2Type mismatch between node output and downstream input fails closed with line-level report.
  3. Step 3Unknown transistor_id or stale version fails per E5.4 graph.
  4. Step 4Integrates with validate-workflow.py in CI on docs/workflows/ changes.
  5. Step 5Pass result stored in state.pursuit.active_workflow.validation_hash.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
  • Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
  • L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
  • Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
  • Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

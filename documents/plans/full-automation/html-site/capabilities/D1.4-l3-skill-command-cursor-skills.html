L3 skill command cursor skills

L3 wraps orchestration into Cursor skills and slash commands under .cursor/skills/ and .cursor/commands/: bounded prompts, allowed tool sets, and phase entry points the conductor invokes by name. Skills sit above scripts—they compose S0 calls and journal-keeper behavior rather than replacing them.

Platform work type D4.3 produces L3 wrappers from stabilized playbooks. Pack consumers inherit skills through template-pack fragments (D1.6); ad hoc skill forks without catalog provenance violate D5.3. Skill promotion should include a smoke verify in SEC-15 release rows.

How it works

  1. Step 1Before guessing URLs, API endpoints, credentials, or team contacts, the conductor or Librarian reads docs/facts/INDEX.md and opens the fact file listed for the topic—external truth lives in facts, not design docs or journal narrative.
  2. Step 2Facts rank lowest in compose (E2.3): agents use facts to disambiguate when scripts, playbooks, and skills do not cover the detail—never to replace executable catalog components.
  3. Step 3Blocking questions about external configuration resolve by reading or extending docs/facts/; if no fact exists, pursuit stops at H2 and requests operator input rather than inventing API shapes in chat.
  4. Step 4Context retrieval via E1.4 and allowed_reads may include fact paths; workers must not expand beyond the indexed files the conductor bound for the turn.
  5. Step 5When upstream systems change and facts go stale, the conductor dual-writes a journal note and may trigger E5.3 reconcile-stale before continuing pursuit on config-dependent steps.

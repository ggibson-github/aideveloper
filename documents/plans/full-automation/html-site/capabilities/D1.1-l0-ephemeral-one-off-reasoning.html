L0 ephemeral one-off reasoning

L0 is the bottom rung of the promotion ladder: reasoning that lives only in the current agent turn—useful once, not yet captured as a reusable artifact. Every pursuit session starts here for novel problems; the platform queue exists so repeated L0 patterns can graduate upward instead of being re-derived every sprint.

L0 work is legitimate product progress but a liability for scale. When the same manual sequence, explanation, or routing choice appears twice, enqueue a promotion item with target_level above L0. Pair with D2.1 enqueue triggers and Vision §6 — Branch D — Platform evolution plane (parallel queue) for the parallel-queue model.

How it works

  1. Step 1On each pursuit wake for novel work, the conductor and workers operate at L0—ephemeral reasoning confined to the current turn until the outcome is captured as a reusable artifact on the promotion ladder.
  2. Step 2L0 progress is valid product delivery but creates scale risk; when the same routing choice, manual sequence, or explanation repeats across turns, pursuit enqueues a state.platform.promotion_queue item with target_level above L0. triggers.
  3. Step 3The autopilot scheduler interleaves platform turns with product work so repeated L0 patterns graduate into playbooks, scripts, or catalog entries instead of being re-derived every sprint.
  4. Step 4The conductor dual-writes enqueue events and current ladder level to state.json and journal so operators can audit what remains ephemeral versus promoted.
  5. Step 5If a pattern recurs twice without promotion enqueue, or L0 reasoning contradicts an existing playbook without reconcile-stale, pursuit stops at H2 until the platform backlog is corrected.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • Platform queue depth 0 but product blocked on missing playbook — D3.3 priority cut skips platform drain.
  • Edge case D1.1 variant 4: verify state dual-write before continuing pursuit.

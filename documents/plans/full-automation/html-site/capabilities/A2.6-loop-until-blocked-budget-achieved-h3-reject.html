A2.6 · Pursuit & control plane

loop until blocked budget achieved H3 reject

loop until blocked budget achieved H3 reject

Pursuit is a loop, not a single shot. This capability states the termination conditions: continue looping through preflight → step → post-step until blocked, budget exhausted, goal achieved, or H3 rejected.

Each iteration is bounded by A2.1–A2.5. The loop may run in IDE autopilot, local SDK daemon, or headless CI depending on Plane I runtime—but the control flow is identical.

Without an explicit loop spec, implementers build ad-hoc while-loops that diverge across runtimes. This document is the canonical cycle for always-on operation.

How it works

  1. Step 1After H1 approves a plan, pursuit enters the canonical always-on loop: A2.1 preflight, then A2.2 one skill phase, then A2.3 post-step dual-write—repeating until a termination condition from the A4 stop taxonomy fires.
  2. Step 2Each iteration stays bounded by A2.4 goal_verify handoff and A2.5 H3-pending transition so scope completion and human final sign-off remain part of the cycle, not bypassed by the loop.
  3. Step 3The loop terminates on check-pipeline-blocked BLOCKED (H2), budget exhaustion., goal.state achieved after H3 acceptance, or H3 rejection that returns pursuit to active work with structured notes.
  4. Step 4The same control flow runs whether the runtime is IDE /autopilot, local SDK daemon, or headless CI—Plane I hosts execution but must not invent alternate while-loop semantics across runtimes.
  5. Step 5If dual-write fails mid-loop or journal and state.json diverge, pursuit halts at H2 before the next wake; resume always restarts from reconciled state at A2.1 preflight.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • goal_verify passes but H3 rejected — goal.state returns to pursuing with rejection notes.
  • Edge case A2.6 variant 4: verify state dual-write before continuing pursuit.

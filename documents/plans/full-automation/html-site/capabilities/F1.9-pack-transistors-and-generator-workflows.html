pack transistors and generator workflows

Pack schema adds workflows/*.json generator templates and transistors/*.json domain blocks—game studio mesh/rig/import, data platform ingest/model/deploy, software delivery scaffold chains.

See Vision §8 — Branch F — Organization plane (template-packs = ceiling) and SEC-18-transistor-model-a-to-z.

How it works

  1. Step 1company.yaml references default_workflow_template per pipeline_id.
  2. Step 2Roles bind to subgraphs of pack workflows via F1.2 pipeline slice fields.
  3. Step 3Pack CI validates all transistors and workflows against v1 schemas.
  4. Step 4Reference implementations in SEC-15-v2.20 and v2.21 include transistor libraries.
  5. Step 5Export upward from consumer repos via D4.6 pack fragment export.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
  • Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
  • L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
  • Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
  • Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

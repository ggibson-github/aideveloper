D4.3 · Platform evolution plane

platform, work, skill, command, and wrapper

platform work skill command wrapper

Skill and command wrapper work packages L2 scripts plus journal-keeper hooks into L3 .cursor/skills/ or /commands entry points—thin orchestration surfaces the conductor invokes by name. Wrappers encode allowed_reads, evidence requirements, and forbidden paths consistent with active role (B5.1).

Wrapper promotion does not duplicate script logic inside SKILL.md; it calls S0 scripts and cites playbook paths. New commands register in operator dashboard docs when user-facing. Failed wrapper verify leaves queue item open; partial SKILL without command or vice versa re-enqueues until both surfaces exist or pack policy declares skill-only.

How it works

  1. Step 1When the promotion queue targets skill or command wrapper work, an economy worker packages promoted L2 scripts plus journal-keeper hooks into .cursor/skills/ or command entry points the conductor invokes by name.
  2. Step 2Wrapper surfaces call S0 scripts by path, cite playbook locations, and encode allowed_reads, evidence requirements, and forbidden paths consistent with B5.1—never duplicating script logic inside SKILL.md.
  3. Step 3User-facing commands register in operator dashboard docs when pack policy requires operator discoverability.
  4. Step 4The verifier runs the promotion verify pattern; on pass the conductor dequeues the item and product turns may compose the new surface via B4.3.
  5. Step 5Failed wrapper verify, a partial SKILL without command, or pack binding mismatch leaves the item open—re-enqueue until both surfaces exist or pack policy declares skill-only.

G6.4 · Verification & quality plane

workflow, checkpoint, replay, from, failed, and node

workflow checkpoint replay from failed node

On node verify failure, active_workflow records failed_node_id and checkpoint artifacts; resume replays from that node after H2 fix—not full workflow restart unless graph changes.

See Vision §9 — Branch G — Verification & quality plane (anti-mistake) and SEC-18-transistor-model-a-to-z.

flowchart TD
  compose[Compose workflow] --> validate[validate-workflow-dag S0]
  validate -->|pass| exec[Execute one node]
  exec --> nodeVerify[Transistor verify]
  nodeVerify -->|pass| next[Advance active_workflow]
  nodeVerify -->|fail| h2[H2 or retry gate]
  next --> goalVerify[goal_verify rollup]

How it works

  1. Step 1state.pursuit.active_workflow.failed_node_id set; completed_nodes preserved.
  2. Step 2Retry increments node retry_count; gate edges handle retry vs H2 per B6.4.
  3. Step 3Graph edit invalidates downstream completed nodes via E5.4 stale mark.
  4. Step 4preCompact snapshot H6 includes active_workflow for autopilot resume.
  5. Step 5Manual operator reset requires journal entry clearing failed state.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
  • Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
  • L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
  • Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
  • Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

platform work playbook-keeper

Playbook-keeper is the primary L1 platform work type: distill repeated reasoning, divergence log entries, or task post-mortems into docs/playbooks/<slug>.md with INDEX registration planned. Economy platform workers run under conductor contract with queue item id; they do not edit consumer feature scope or dual-write pursuit state.

Output feeds script extraction (D4.2) when steps stabilize. Playbook-keeper follows compose-first (B4.3)—extend existing playbooks before new slugs. Completion requires playbook path, word-quality bar, and enqueue downstream promotion if L2 candidate steps are identified.

How it works

  1. Step 1When platform scheduling dequeues a promotion item typed playbook-keeper, conductor spawns an economy platform worker bound to the queue item id with scoped allowed_reads—no permission to edit consumer feature scope or dual-write product pursuit state.
  2. Step 2The worker distills repeated reasoning, divergence log entries, or task post-mortems into docs/playbooks/<slug>.md, following B4.3 compose-first by extending existing INDEX playbooks before creating new slugs.
  3. Step 3Completion dual-writes the playbook path into the promotion queue item, passes word-quality bar checks, and plans INDEX registration for the new or extended entry.
  4. Step 4When steps in the playbook stabilize as L2 candidates, the worker enqueues downstream D4.2 promotion rather than marking the queue item done prematurely.
  5. Step 5If the worker edits journal or consumer state.json, produces prose below the quality bar, or marks done without a playbook path, pursuit fails closed at H2 and the promotion item reopens.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • Platform queue depth 0 but product blocked on missing playbook — D3.3 priority cut skips platform drain.
  • Edge case D4.1 variant 4: verify state dual-write before continuing pursuit.

A6.3 · Pursuit & control plane

operator, observe, without, unblocking, and loop

operator observe without unblocking loop

Operators may inspect dashboards, journals, evidence logs, and queue depth while pursuit remains blocked without that inspection counting as “unblocking.” Read-only observation must not clear H2, advance next_action, or waive gates.

This separation prevents accidental resume when an operator opens STATUS to diagnose a failure. Explicit unblock actions—supplying credentials, answering a blocking question, approving H3—are distinct events recorded in journal Q&A. Autonomous loops remain paused until preflight passes on the actual fix.

How it works

  1. Step 1Operators may read A6.1 dashboard, STATUS.md, journal/progress.md, evidence logs, and platform queue depth while pursuit remains BLOCKED—read-only inspection never clears H2, advances next_action, or waives gates_pending.
  2. Step 2A5.1 and A5.2 treat observation separately from resume: opening STATUS to diagnose a failure does not count as Continue or implicit gate approval.
  3. Step 3Explicit unblock actions—supplying credentials, answering a blocking_questions entry, recording H3 approval or rejection—are distinct journal Resolved Q&A events that preflight must see before A2.2 may run again.
  4. Step 4Under A3.2 and A3.4 unattended modes, the loop stays paused at A4.1 H2 until preflight passes on the actual fix, not when the operator merely acknowledged a digest.
  5. Step 5If an operator action is ambiguous (e.g. a comment without answering the blocker), pursuit dual-writes H2 with the unresolved question—observation alone never transitions goal.state from blocked to pursuing.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • goal_verify passes but H3 rejected — goal.state returns to pursuing with rejection notes.
  • Edge case A6.3 variant 4: verify state dual-write before continuing pursuit.

runtime IDE local subagents economy

Local subagents run as economy-tier Task/explore/shell workers under conductor contracts with capped allowed_reads. The IDE runtime spawns them via orchestrate-subagents; workers return summaries without touching journal or state.json.

Parallel local subagents are allowed only when integration manifest declares independent lanes (C4.2). Model tier comes from route-tier.py output, not ad-hoc per spawn. Failed worker runs log to H5 audit trail.

How it works

  1. Step 1When spawn_workers is true and the conductor needs bounded context, orchestrate-subagents spawns economy-tier Task, explore, or shell workers with contracts that cap allowed_reads and forbid journal or state.json writes.
  2. Step 2Before each spawn, route-tier.py reads state.json and sets model_tier so workers run at economy tier—not ad-hoc model picks per invocation.
  3. Step 3Parallel local subagents start only when the integration manifest lists independent lanes.; otherwise the conductor serializes worker turns.
  4. Step 4Workers return summaries to the parent; the conductor alone merges results and dual-writes journal and state.json after the pipeline step completes.
  5. Step 5Failed or timed-out worker runs append to H5; persistent failures surface as H2 blockers without marking the pursuit step complete.

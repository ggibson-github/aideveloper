data platform roles analyst engineer DBA SRE governance

The data platform example pack defines roles: analyst, data engineer, DBA, SRE, governance reviewer—each with pipeline bindings for ingest-to-production workflows. Roles demonstrate regulated data work: schema approval, deploy windows, and audit evidence distinct from game asset roles (F3.1).

Governance role may hold H2-equivalent gates inside pack policy (strict_hitl). Tool permissions restrict production deploy commands to SRE role (F6.3). Pipelines (F4.2) sequence work across these roles. Shared micro-packs (F5.1) may add generic security review fragments.

How it works

  1. Step 1When a data-platform pack goal activates, F2.2 maps ingest, model/transform, deploy, and monitor phases to engineer, DBA, and SRE lanes with governance checkpoints before parallel work starts.
  2. Step 2Before deploy lanes dequeue, F1.4 manifest approval and F1.5 artifact-graph edges must show schema sign-off—S0 validates pipeline YAML against pack schema so bad graphs never land in state.json.
  3. Step 3Deploy-phase pursue steps require evidence bundles per pack policy; the conductor blocks deploy outside approved maintenance windows until H1 waives or the window opens.
  4. Step 4Monitor-phase outputs feed F1.8 metrics checks so goal_verify aggregates pipeline health without subjective review.
  5. Step 5If schema validation fails, manifest edges are missing, or deploy evidence is incomplete, pursuit fails closed at H2—lanes do not advance to deploy or monitor until proof is attached.

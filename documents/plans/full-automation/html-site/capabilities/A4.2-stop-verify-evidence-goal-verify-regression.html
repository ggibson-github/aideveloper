A4.2 · Pursuit & control plane

stop, verify, evidence, goal verify, and regression

stop verify evidence goal_verify regression

Verification stops protect quality when evidence or goal-level proof fails. Task evidence missing, verify-router failure, goal_verify regression, or conformance script exit non-zero all produce a hard stop with structured H2—not a silent retry loop.

Fail-closed behavior prevents false “achieved” states that would reach H3 without proof. The conductor records which command failed and which evidence paths were absent so the operator or a resumed pursuit knows exactly what to fix. This stop class connects Plane A to Plane G; it is distinct from resource exhaustion (A4.3) and integrity corruption (A4.4).

How it works

  1. Step 1When preflight finds missing task evidence (evidence_required true but paths absent), G1.1 or verify-router exit non-zero, goal_verify_command failure, or G2.4 regression, check-pipeline-blocked emits stop class verify:* and exits the A2.6 loop without advancing next_action.
  2. Step 2Journal-keeper dual-writes the failed command, missing evidence paths, and last_verify=failed to journal and state.json so resumed pursuit or the operator knows exactly what to fix—fail-closed, no silent retry loop.
  3. Step 3Task-level verify failures block implement task completion; goal-level failures block A2.5 H3 pending transition., preventing false achieved states that would reach H3 without proof.
  4. Step 4This stop class connects Plane A pursuit to Plane G verification—it is distinct from resource exhaustion (A4.3) and state corruption (A4.4).
  5. Step 5If verify stops fire but evidence later passes on re-run, pursuit resumes only after preflight confirms all required proof paths exist—never auto-clearing H2 without successful verifier output.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • goal_verify passes but H3 rejected — goal.state returns to pursuing with rejection notes.
  • Edge case A4.2 variant 4: verify state dual-write before continuing pursuit.

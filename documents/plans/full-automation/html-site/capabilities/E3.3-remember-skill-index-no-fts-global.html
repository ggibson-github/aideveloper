remember skill INDEX no FTS global

The remember skill captures operator snippets into docs/facts/captured.md and updates facts INDEX—explicit "remember this" workflow, not ambient chat memory. There is no global FTS or hidden vector store; all retained knowledge is file-backed and discoverable via E1.4.

This keeps Plane E honest: if it is not in facts/decisions/catalog, agents should not treat it as project truth. Remember complements journal Resolved Q&A for durable non-decision snippets (Slack URLs, SQL fragments). Promotion candidates from repeated captures may enqueue via E2.5.

How it works

  1. Step 1When evidence_required is true in state.json. verify-router for each implement task and blocks advancing next_action until last_verify=passed and evidence paths exist on disk.
  2. Step 2Workers may not mark TODOS or task cards complete in prose—the conductor dual-writes completion only after verify-router exit 0 and evidence files are recorded in journal and state.
  3. Step 3S0 validate-workflow.py checks last_verify and evidence_files on every wake; mismatch between claimed done and machine state triggers an integrity stop.
  4. Step 4Audit and hierarchy quality scripts re-enqueue deepen when queue items close without artifacts, preventing silent hallucinated done from persisting across passes.
  5. Step 5If pursuit would advance without demonstrated evidence, the conductor fails closed at H2 rather than assuming success—Plane G treats demonstrated verify as the only valid completion signal.

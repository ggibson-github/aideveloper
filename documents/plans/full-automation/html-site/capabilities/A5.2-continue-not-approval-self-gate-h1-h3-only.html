continue not approval self-gate H1 H3 only

Continue is not approval. Typing Continue or running /continue does not waive HLD, DD, feature design, or manifest gates—those become self-gate plus evidence where automatable, with H1 and H3 remaining the only default human approvals unless strict HITL is enabled.

Misreading Continue as “yes, proceed anyway” reintroduces silent gate skips. The conductor and S0 scripts treat unresolved blocking questions and pending human gates as BLOCKED preflight outcomes. Only explicit approval phrases or recorded waivers in the journal clear design gates.

How it works

  1. Step 1When the operator types Continue or /continue, A2.1 checks gates_pending, blocking_questions, and unresolved HLD/DD/feature-design waits—treating them as BLOCKED. even if other fields look READY.
  2. Step 2Continue does not waive HLD, DD, manifest, or feature-design gates; automatable design checks run as self-gate plus evidence, leaving H1 plan approval and H3 sign-off as the default human touchpoints.
  3. Step 3Only explicit approval phrases (e.g. "I approve the HLD") or recorded waivers in journal Resolved Q&A clear design gates—A5.1 never interprets silence or repeated Continue as consent.
  4. Step 4S0 scripts in route-tier and validate-workflow enforce the same rule headlessly so A3.4 daemons cannot skip gates because no human watches the chat.
  5. Step 5If an operator believed Continue cleared a gate but preflight still blocks, pursuit dual-writes H2 with the pending gate id and re-prompts for substantive approval—not another pipeline phase.

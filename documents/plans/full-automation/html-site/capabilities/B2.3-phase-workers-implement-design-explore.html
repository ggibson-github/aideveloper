Phase workers implement design explore

Phase workers implement one skill phase under contract: implement-feature, dd-writer, explore, refactor—each with task card paths, evidence requirements, and no journal writes. The conductor selects the worker type from next_action and route-tier output.

Parallel phase workers are allowed only when the integration manifest declares independent lanes; otherwise one worker per turn preserves auditability. Workers return summaries; the conductor merges and verifies before marking progress.

How it works

  1. Step 1From next_action, the conductor spawns one phase worker type per turn—implement-feature, dd-writer, librarian explore, refactor—each under a task card or skill contract with evidence requirements and no journal writes.
  2. Step 2Parallel phase workers run only when the integration manifest declares independent lanes; otherwise one worker per turn preserves auditability and avoids conflicting dual-write.
  3. Step 3Workers execute the scoped phase, return summaries and evidence paths, and must not choose the next pipeline step or update journal/state.json themselves.
  4. Step 4The conductor verifies worker output against task card acceptance criteria before marking phase progress or spawning B2.4 for command proof.
  5. Step 5If a worker expands beyond allowed_reads, omits required evidence, or attempts journal writes, pursuit blocks at H2 until the conductor rescopes or respawns under a corrected contract.

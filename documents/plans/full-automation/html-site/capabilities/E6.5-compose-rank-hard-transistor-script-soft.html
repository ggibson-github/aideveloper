compose, rank, hard, transistor, script, and soft

Extends E2.3 ranking for workflow node binding: hard_transistor > script > soft_transistor > playbook > skill > facts. Gate transistors are not capability substitutes—they are edges.

See Vision §7 — Branch E — Knowledge & composition plane and SEC-18-transistor-model-a-to-z.

How it works

  1. Step 1When filling a workflow node, composer applies E6.5 before E2.3 legacy single-component compose.
  2. Step 2Ties break toward higher maturity and pack-authored transistors (F1.9).
  3. Step 3Invoking soft when hard exists requires B4.4 divergence with justification.
  4. Step 4Facts disambiguate only; never replace executable transistors in ranked slots.
  5. Step 5Conformance check in validate-workflow.py when task card transistor_id mismatches rank.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
  • Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
  • L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
  • Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
  • Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

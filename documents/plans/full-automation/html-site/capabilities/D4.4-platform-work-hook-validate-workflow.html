platform work hook validate-workflow

Hook and validate-workflow extension is L4 platform work: extend .cursor/hooks.json or CI steps so validate-workflow, check-pipeline-blocked, or staleness scripts run on session events without conductor turns. Changes must pass python scripts/validate-workflow.py on the PR that introduces them.

Hooks are not pursuit loops—they fail closed into journal notes when scripts error. Extend-only hook changes follow D5.2; hook behavior that alters gate semantics needs explicit pack review. Successful promotion moves ambient enforcement up the D1.5 ladder so S0 scripts run even when the conductor forgets.

How it works

  1. Step 1On L4 hook promotion items, a platform worker extends .cursor/hooks.json or CI steps so validate-workflow, check-pipeline-blocked, or staleness scripts fire on session events without a conductor turn.
  2. Step 2Introduced hooks must pass python scripts/validate-workflow.py on the same PR; hook script errors fail closed into journal notes rather than silently continuing pursuit.
  3. Step 3Successful promotion advances ambient enforcement on the D1.5 ladder so S0 scripts run even when the conductor omits them.
  4. Step 4Extend-only hook changes follow D5.2; hook edits that alter gate semantics require explicit pack review and may escalate H2.
  5. Step 5If hooked scripts error or alter H1/H2/H3 semantics without approval, pursuit stops with an integrity stop. until hooks are repaired or reverted.

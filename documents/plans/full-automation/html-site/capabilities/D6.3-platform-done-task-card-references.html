D6.3 · Platform evolution plane

platform, done, task, card, and references

platform done task card references

At least one task card—or playbook procedure consumed by task cards—must reference the promoted artifact so future implement turns exercise it under evidence gates. Orphan scripts unused by pursuit are platform debt, not maturity.

Reference can be Components field, Test command, or explicit allowed_reads entry on a template task card. Playbook-keeper should add reference guidance when capturing patterns from a source task. D6.3 closes the loop from product repetition to reusable tooling back to product—without reference, promotion did not enter the delivery loop and C3.1 promotion_note intent is unfulfilled.

How it works

  1. Step 1Before marking platform promotion done, audit confirms at least one task card—or playbook procedure consumed by task cards—references the promoted artifact in Components, Test command, or allowed_reads.
  2. Step 2Playbook-keeper adds reference guidance when capturing patterns from a source task so future implement turns exercise the tool under evidence gates.
  3. Step 3Orphan scripts unused by pursuit are platform debt; promotion without a reference fails D6.3 and reopens the queue item.
  4. Step 4Reference closes the loop from product repetition to reusable tooling back to product. promotion_note intent.
  5. Step 5If no task card or playbook procedure cites the artifact, pursuit blocks platform done at H2 until a template or active card is updated and dual-written.

L2 script S0 scripts/

L2 moves repeatable behavior into S0 scripts under scripts/: deterministic, testable, and mandatory-first per B1.1. Script extraction (D4.2) promotes playbook steps that no longer need model judgment—routing, validation, queue peek, verify dispatch.

Each L2 artifact needs a unit test path and, when it replaces manual verify, a verify-router hook (D6.2). The conductor runs scripts before improvising; workers must not re-implement script logic in chat. Failed promotion leaves the queue item open with H2 packaging, not a false done mark.

How it works

  1. Step 1L2 promotion moves repeatable playbook steps into deterministic scripts under scripts/—routing, validation, queue peek, and verify dispatch. script extraction on platform turns.
  2. Step 2At pursuit wake, B1.1 mandatory-first requires the conductor to run applicable L2 scripts and use their output before any worker re-implements the same logic in chat.
  3. Step 3Each promoted L2 script carries a unit test path under tests/unit/ and, when it replaces manual verify, registers with G1.1 verify-router.
  4. Step 4Platform workers complete L2 promotion only when the script exists, tests pass, and catalog or playbook references point to the script—dual-write promotion_queue completion to journal and state.json.
  5. Step 5If script extraction marks done without a test, verify-router cannot dispatch the script, or workers duplicate script behavior inline, pursuit fails closed at H2 with the promotion item reopened—never false-complete L2 packaging.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • Platform queue depth 0 but product blocked on missing playbook — D3.3 priority cut skips platform drain.
  • Edge case D1.3 variant 4: verify state dual-write before continuing pursuit.

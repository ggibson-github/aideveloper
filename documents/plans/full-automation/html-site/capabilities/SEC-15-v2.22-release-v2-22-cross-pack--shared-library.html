Release v2.22 cross-pack shared library

v2.22 release v2 22 cross pack shared library

Release v2.22 adds template-packs/_shared/ for cross-pack libraries—common roles, hooks, and verification helpers factored out of duplicate pack content. Packs should compose shared fragments rather than fork copy-paste.

Catalog entries should reference _shared components explicitly.

flowchart TD
  trigger[Trigger condition] --> pre[Preconditions S0]
  pre --> exec[Execute step logic]
  exec --> verify[Verify output]
  verify -->|pass| done[Mark queue item done]
  verify -->|fail| h2[H2 blocker]

What this defines

  1. Step 1When policy, pack authority, or unresolved external dependency requires human judgment beyond B1.4 self-gate, B2.1 classifies the turn as S4 and stops pursuit before another A2.2 phase runs.
  2. Step 2The conductor packages an H2 blocker with goal_id, missing artifact or authority, typed stop class., and a concrete operator action—never a vague “need help.”
  3. Step 3Waiver requests, legal/finance pack gates, and multi-goal conflicts dual-write to journal Blockers and state.json hitl fields so A2.1 returns BLOCKED until cleared.
  4. Step 4A6.2 emits a digest notification on S4 escalation; pursuit does not busy-loop waiting for operator chat acknowledgment.
  5. Step 5If S4 is invoked for work resolvable by S0/S1/S3, preflight flags mis-routing at H2—S4 is reserved for imminent H2 or policy-mandated review, not default genius-tier convenience.

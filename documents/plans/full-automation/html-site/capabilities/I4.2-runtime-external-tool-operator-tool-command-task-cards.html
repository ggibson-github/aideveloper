runtime external tool-operator Tool command task cards

Tool-operator shell workers execute literal Tool command blocks from task cards—Blender batch exports, Unreal commandlet runs, custom CLIs—writing outputs only to declared paths. tool-operator skill bounds writes; product source edits happen only via normal implement workers.

Tool failures attach logs to evidence/ and trigger verify escalation (B2.4). Pack authors declare tool patterns in task card templates; ad-hoc tool invocation without card binding is forbidden during evidence-gated implement.

How it works

  1. Step 1When an implement task card declares a Tool command block, the conductor spawns a tool-operator shell worker instead of a phase implement worker, binding execution to the literal CLI on the card.
  2. Step 2The tool-operator worker runs pack-declared commands—Blender batch exports, Unreal commandlets, custom CLIs—and may write outputs only to paths explicitly declared on the task card.
  3. Step 3tool-operator skill bounds the command surface and filesystem writes; product source edits remain the domain of normal implement workers, not shell tool runs.
  4. Step 4On completion, stdout/stderr and artifact paths attach to evidence/ and route through B2.4 before last_verify can pass and pursuit advances.
  5. Step 5Tool failures, missing outputs, or ad-hoc invocations without a bound task card fail closed with verify escalation—pursuit turns stop rather than improvising undocumented tooling.

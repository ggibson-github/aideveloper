A2.5 · Pursuit & control plane

goal verify pass transition H3 pending

goal_verify pass transition H3 pending

Passing goal_verify is necessary but not sufficient for closure. This capability defines transition to H3 pending: verification succeeded, human final sign-off is required, and the goal state reflects verifying → awaiting H3.

The system notifies the operator; it does not block the world indefinitely unless strict HITL mode demands it. H3 is the third human touchpoint—accept or reject verified outcomes. Rejection returns the goal to pursuing with structured notes.

Automating everything except H3 is the INTRO contract; this step enforces that boundary in the pursuit loop.

How it works

  1. Step 1When A2.4 reports goal_verify success, the conductor sets goal.state to verifying and adds H3 sign-off to gates_pending.—autonomous pursuit stops before declaring achieved.
  2. Step 2Journal-keeper dual-writes hitl.h3_pending, verification evidence paths, and last_verify passed to journal/progress.md and state.json in the same update so resume after crash still shows awaiting H3.
  3. Step 3A6.1 surfaces the verified outcome to the operator via dashboard or webhook digest without requiring another implement turn—non-blocking unless strict HITL mode demands a hard wait.
  4. Step 4At H3 the operator accepts (goal.state → achieved) or rejects with structured notes, returning pursuit to pursuing and clearing achieved until A2.1 preflight reconciles scope and evidence.
  5. Step 5If goal_verify passes but H3 pending dual-write fails, or gates_pending omits H3 while notification fires, pursuit fails closed at H2 until journal and state.json reconcile—premature achieved is forbidden.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write.
  • Concurrent manual edit to queue JSON — conductor reloads queue each wake; last writer wins with journal note.
  • goal_verify passes but H3 rejected — goal.state returns to pursuing with rejection notes.
  • Edge case A2.5 variant 4: verify state dual-write before continuing pursuit.

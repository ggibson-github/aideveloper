D4.6 · Platform evolution plane

platform, work, pack, fragment, and export

platform work pack fragment export

Pack fragment export is upward promotion: vetted repo artifacts become template-packs/ fragments new instantiations inherit—default skills, pipeline stubs, verify suites, glossary rows. Export is organizational, not a hotfix to one consumer repo.

Export requires pack-author authority or H2 when external pack repo access is missing. Fragments version semver-style; provenance links to source promotion id (D5.3). This is the L5 capstone work type paired with D1.6; consumer goals should reference pack version, not fork fragments inline. Plane F pack verify suites gate publish.

How it works

  1. Step 1On L5 pack-fragment export items, a platform worker promotes vetted repo artifacts into template-packs/ fragments—default skills, pipeline stubs, verify suites, glossary rows—for future company instantiations.
  2. Step 2Export requires pack-author authority; missing external pack repo access escalates H2 rather than forking fragments inline in a consumer goal repo.
  3. Step 3Fragments receive semver-style version stamps and provenance links to the source promotion id.
  4. Step 4Before publish, F1.8 pack verify suites must pass on the fragment slice.
  5. Step 5Consumer goals reference pack version from catalog, not ad-hoc inline forks; failed pack verify or missing provenance leaves the export queue item open at H2.

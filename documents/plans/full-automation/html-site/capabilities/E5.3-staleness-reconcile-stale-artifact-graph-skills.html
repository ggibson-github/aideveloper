staleness reconcile-stale artifact-graph skills

reconcile-stale and reconcile-artifact-graph skills plan reruns or operator approval when staleness.json or program artifact graphs flag stale nodes. Conductor invokes before implement spawn on affected paths; workers pause until reconciliation clears flags or H2 waiver records.

S0 scripts mark stale; skills decide rerun vs waive vs deepen hierarchy queue. Pair with C4.4 for program mode and E5.1 for design graph source. Silent implement against stale design violates Plane E contract and undermines goal_verify evidence.

How it works

  1. Step 1When staleness.json or a program artifact graph flags stale nodes on paths in the next implement scope, the conductor invokes reconcile-stale or reconcile-artifact-graph before spawning workers—not after partial edits land.
  2. Step 2S0 scripts maintain stale flags file-derived; the reconcile skills decide whether to rerun the affected design phase, waive with H2 operator approval, or enqueue hierarchy deepen items for impacted ids.
  3. Step 3Workers on affected paths pause while reconciliation runs; pursuit resumes only after flags clear or an explicit H2 waiver is dual-written to journal and state.json.
  4. Step 4In program mode, reconcile-artifact-graph consults C4.4 cross-stream contracts alongside E5.1 as the design-graph source of truth.
  5. Step 5If implement spawn would proceed against stale design without reconciliation or waiver, pursuit halts at H2—silent implement violates Plane E contract and invalidates goal_verify evidence.

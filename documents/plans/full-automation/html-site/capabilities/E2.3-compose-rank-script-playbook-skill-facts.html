E2.3 · Knowledge & composition plane

compose, rank, script, playbook, skill, and facts

compose rank script playbook skill facts

Compose ranking orders catalog hits: script (S0) first, then playbook, then skill, then facts. v2.24+ extends this via E6.5: hard_transistor > script > soft_transistor > playbook > skill > facts.

Ranking applies after E2.2 list-components returns hits. Ties break toward higher maturity and pack-authored fragments. Divergence from ranked choice requires B4.4 log entry. Ranking aligns with capability_class S0–S4 in Plane B.

How it works

  1. Step 1After E2.2 list-components returns hits, compose ranks them script (S0) first, then playbook, then skill, then facts—maximizing determinism before any worker improvises procedure.
  2. Step 2Ties break toward higher catalog maturity and pack-authored fragments so template-pack reuse wins over ad-hoc L0 reasoning per company policy.
  3. Step 3The conductor or librarian loads the top-ranked component path into worker allowed_reads; invoking a skill when an S0 script covers the same behavior violates B4.3 unless B4.4 records divergence.
  4. Step 4Facts from docs/facts/ disambiguate ambiguous hits only—they do not replace executable scripts, playbooks, or skills in the ranked ladder aligned with capability_class S0–S4.
  5. Step 5If ranking yields no executable match or the conductor skips the ranked choice without a divergence log, pursuit stops at H2 rather than composing from model memory.

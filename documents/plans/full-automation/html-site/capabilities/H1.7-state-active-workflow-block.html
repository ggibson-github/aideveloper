H1.7 · Persistence & state plane

state, active workflow, and block

state active_workflow block

The state.pursuit.active_workflow block is how the harness remembers where it is in a generator DAG across autopilot turns, laptop closes, and preCompact snapshots. Additive fields include workflow_id, path to docs/workflows/<goal-id>.json, current_node_id, completed_nodes[] (each with evidence paths and transistor version), failed_node_id, per-node retry_counts, optional branches[] for parallel lanes (C6.4), validation_hash from the last validate-workflow-dag.py pass, and terminal_nodes[] for rollup checks (C6.5).

Product workers read this block; only the conductor dual-writes advances after node verify. validate-workflow.py schema-validates the shape on load. sync-state.py repairs partial writes from the journal Workflow section after crash. Null active_workflow is allowed only before C6.1 binds a graph or for taxonomy-exempt goals. Headless SDK consumers read the same block per J5 export contract.

See Vision §19 — Transistor & generator workflow model and APP-B-state-json-sketch for the full additive state map.

How it works

  1. Step 1validate-workflow.py schema-validates active_workflow when present.
  2. Step 2Dual-write on every node completion; product workers read-only except conductor merge.
  3. Step 3Null active_workflow allowed only before C6.1 or for taxonomy-exempt goals.
  4. Step 4sync-state.py repairs partial writes from journal Workflow section.
  5. Step 5Export contract I5/J5 documents active_workflow for headless SDK consumers.

Edge cases

  • Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
  • Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
  • L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
  • Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
  • Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

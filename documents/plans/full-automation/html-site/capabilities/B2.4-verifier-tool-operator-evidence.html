Verifier tool-operator evidence

Verifier and tool-operator are shell-class workers for bounded CLI runs: pytest via verify-router, Blender/Unreal export commands, checksum tools—anything with a literal command block on the task card. They write evidence logs under evidence/ and set last_verify in state.

Non-pytest evidence types follow docs/operator/evidence-types.md. Verifier failure blocks task completion and may trigger B3.3 escalation without clearing gates. Tool-operator never edits product source except via declared tool output paths.

How it works

  1. Step 1When a task card declares a Test command or Tool command, the conductor spawns Verifier or tool-operator shell workers to run the literal CLI block and capture output under evidence/.
  2. Step 2verify-router.py dispatches pytest and records last_verify in state.json; non-pytest evidence follows docs/operator/evidence-types.md paths declared on the card.
  3. Step 3Verifier and tool-operator workers do not edit product source except through declared tool output paths; they return bounded logs to the conductor for merge.
  4. Step 4last_verify=passed is required before the conductor marks an evidence-required implement task complete or advances past implement-feature.
  5. Step 5Verifier failure blocks task completion and may invoke B3.3; workers cannot self-clear evidence gates without conductor re-run.

Work taxonomy build

build work taxonomy build

Build work is implement, integrate, and refactor—the core product execution surface Plane C owns. Tasks must carry evidence commands; pursuit continues only when verify passes or structured H2 fires.

Build roles in packs should default to S0/S1-heavy routing with conductor merge at S3 boundaries.

  4. Step 4Refactor work under the Build taxonomy follows C2.4 phase ordering—tests green before routing marks the slice complete toward C3.4 goal rollup.
  5. Step 5If build tasks lack evidence commands or verify fails without H2 packaging, A2.1 blocks further implement turns until task cards and state.json reconcile.

workstreams lane.json leases

Workstreams expose lane JSON with lease fields so parallel program lanes claim work safely. Leases expire per A4.3 resource stops; stale leases return items to queue with journal note.

Lane files live under docs/manifest/ program paths; integration-manifest-keeper maintains cross-lane contracts. One conductor remains writer for journal/state even when lanes parallelize implement.

How it works

  1. Step 1During pursuit, when the conductor or a worker runs the identical manual shell command twice—same pytest flags, export script, or validate call—S0 observation records the duplicate and enqueues a promotion candidate into state.platform.promotion_queue instead of tolerating endless repetition.
  2. Step 2The trigger requires observed duplicate invocation, not speculation after one run; the new item carries source task or journal context, suggested target_level (often L2 script), and priority. schema.
  3. Step 3Enqueue does not block the current product turn—the project schedule continues under D3.1 while platform workers drain the head on dedicated platform turns.
  4. Step 4When D3.3 signals product blocked on missing tooling, the scheduler may invert priority to favor platform drain before the next implement turn.
  5. Step 5If duplicate detection fires without journal provenance, schema validation fails, or enqueue races concurrent manual queue edits, pursuit fails closed at H2 until the conductor dual-writes a conformant queue item and records the duplicate fingerprint.

## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. Before any shell or tool invocation, preToolUse hooks in `.cursor/hooks.json` evaluate the proposed command against role allowlists from [B5.2](B5.2-role-to-pipeline-id-skills-tool-permissions.md) and active pack policy—deny is fail-closed with a journal note, not silent strip.
2. Tool-operator and verifier workers run only bounded literal commands listed on the active task card; economy search and explore workers must not escalate to destructive or unapproved system commands.
3. When a worker proposes a command outside the card or role permission set, the conductor rejects the invocation before merge rather than accepting out-of-allowlist shell access.
4. Forbidden patterns—destructive file ops, credential exfiltration, disallowed network calls—block pursuit immediately at the hook layer without advancing implement or verify turns.
5. If an unsafe command is attempted or a hook deny is overridden in chat, pursuit fails closed at H2 with command evidence attached until the operator approves a bounded alternative or updates pack policy at H1.

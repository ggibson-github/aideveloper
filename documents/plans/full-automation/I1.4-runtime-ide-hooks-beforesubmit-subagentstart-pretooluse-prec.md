## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. On continue/start, beforeSubmit and subagentStart hooks in `.cursor/hooks.json` run S0-thin scripts that inject journal summary, state.json snapshot fields, and context file paths into the conductor turn.
2. preToolUse hooks enforce allowlisted commands and filesystem writes before economy workers execute shell or MCP tooling during implement tasks.
3. preCompact fires sync-state snapshot writes paired with [H6](H6-snapshots.md) so pursuit can resume after context compaction without losing the last good dual-write baseline.
4. Hooks complement always-on rules from [E4.1](E4.1-context-always-on-rules-agents-md.md) but do not replace the continue skill requirement to read journal/progress.md each wake.
5. If hook injection fails—corrupt state.json, missing journal, or script error—pursuit surfaces H2 and refuses the turn rather than running the conductor on stale or empty context.

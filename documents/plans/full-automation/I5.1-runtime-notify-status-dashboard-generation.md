## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. After journal-keeper dual-write at session end or phase completion, the conductor runs generate-dashboard ([B1.1](B1.1-s0-deterministic-mandatory-first.md) script or skill) to regenerate `STATUS.md` and `docs/operator/dashboard.md` from `state.json` and the staleness manifest.
2. Dashboard rows surface goal_id, `next_action`, blockers, and evidence status so operators can resume pursuit without opening the full journal.
3. Dashboard output is observational only per [A6.3](A6.3-operator-observe-without-unblocking-loop.md)—regeneration does not advance `next_action`, clear H2, or substitute for H3 sign-off.
4. IDE, SDK, and headless runtimes share the same S0 regeneration path so operator snapshots stay consistent regardless of where pursuit runs.
5. If `state.json` or staleness inputs are unreadable, regeneration fails closed and the conductor records the fault rather than publishing stale or empty dashboard rows.

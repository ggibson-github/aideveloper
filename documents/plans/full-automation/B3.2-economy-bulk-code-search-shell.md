## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. Economy tier handles bulk mechanical work—multi-file search, repetitive task-card refactors, test fixes with clear stack traces, catalog indexing—under stricter read/write contracts than the conductor.
2. Bulk code search and shell subagents operate within `allowed_reads`; findings return as summaries for S3 escalation when architectural conflict or missing facts appear outside the contract.
3. Economy tier is narrower context and narrower write scope, not lower quality by default; the conductor escalates to [B1.4](B1.4-s3-architecture-ambiguity.md) when search surfaces design conflicts.
4. After economy workers finish, verify and merge follow the same evidence discipline as S1 implement workers before `next_action` advances.
5. If economy workers write outside task-card paths or skip returning bounded summaries, pursuit stops at H2 with scope creep per [G5.2](G5.2-mistake-scope-creep-allowed-reads-task-card.md).

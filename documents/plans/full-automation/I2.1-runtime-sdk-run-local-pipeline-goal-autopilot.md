## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When `goal_autopilot` is active, `scripts/automation/run-local-pipeline.py` wakes on schedule and runs the continue skill without an operator typing `/continue`, reading `next_action` from `journal/state.json` under the same contract as IDE pursuit ([A3.4](A3.4-sdk-daemon-run-local-pipeline-24-7.md)).
2. Before each daemon iteration, S0 `check-pipeline-blocked` runs; gates, blocking questions, budget caps, or corrupt state stop the loop and surface H2 instead of advancing pursuit.
3. Each iteration spawns Cursor agents or SDK equivalents per `docs/operator/model-policy.json`—genius-tier conductor for orchestration, economy workers for bounded tasks—while dual-writing journal and state after every step.
4. When in-scope work is complete, the daemon routes to [goal_verify](A1.3-goal-verify-command-meta-test.md) rather than another implement task, then holds for H3 sign-off if verification passes.
5. Worker run metadata feeds [H5](H5-worker-runs.md) audit; on integrity validation failure or repeated verify failure, autopilot halts with structured H2 and `last_failure` recorded in the journal.

## Reader narrative

S1 covers mechanical work that still needs a model but not genius-tier judgment: bounded code edits from task cards, catalog lookups, shell verification, and bulk search. Economy workers from `model-policy.json` execute S1 under conductor contracts with `allowed_reads` caps.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When route-tier classifies a turn as S1 and `spawn_workers` is true, the conductor spawns economy-tier implement workers from `model-policy.json` with task-card scope and `allowed_reads` caps—never genius models for mechanical bounded edits.
2. S1 workers execute catalog lookups, shell verification, and bulk search under orchestrate-subagents contracts and must not expand scope, dual-write journal/state, or mark tasks complete without evidence paths.
3. The conductor selects worker roles using Plane E catalog component refs so recurring implement patterns map to verifier or tool-operator roles instead of improvising role choice each wake.
4. [B1.1](B1.1-s0-deterministic-mandatory-first.md) scripts remain mandatory before S1 workers run when file-derived routing, validation, or verify already exists in `scripts/`.
5. Mis-routed S3 architecture work assigned to S1, or S1 tasks escalated to genius without `model_escalation`, stops at H2 until route-tier re-classifies capability and dual-writes corrected `model_tier`.

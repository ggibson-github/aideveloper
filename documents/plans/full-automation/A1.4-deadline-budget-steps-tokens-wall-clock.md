# A1.4: deadline budget steps tokens wall clock

**Parent:** [A1-index](A1-index.md) · **Branch A** · **Vision §3** · **Release:** v2.14

## Reader narrative
<!-- prose-source: agent pilot 2026-06-28 -->

Autonomous pursuit without bounds can consume unbounded tokens, steps, and wall-clock time. This capability defines deadline and budget fields—steps, tokens, wall clock—so the loop can stop cleanly with a recorded reason instead of running indefinitely.

Budgets are enforced at preflight and after each turn. When a budget is exhausted, pursuit stops, dual-writes a structured stop reason, and surfaces H2 only if human intervention can extend the budget. Deadlines align with operator expectations for time-sensitive milestones.

These fields turn “always-on” into “always-on until provably done or bounded.” They also inform Plane D scheduling so self-improvement work stays bounded and does not displace the current project goal. Incorrect budget configuration is an operational edge case: too tight causes false stops; too loose wastes resources.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. At H1 plan approval, the operator sets deadline and budget fields—max_steps, max_tokens, and wall-clock deadline—in state.goal alongside [A1.2](A1.2-success-criteria-machine-checkable.md) success criteria so [A3.2](A3.2-goal-autopilot-until-goal-verify-or-hard-block.md) pursuit is bounded instead of unbounded.
2. On each wake, [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) compares live counters against those caps before routing; an exhausted budget or passed deadline fires an [A4.3](A4.3-stop-resource-max-steps-max-cost-lease-expired.md) resource stop instead of scheduling another pipeline phase.
3. After [A2.3](A2.3-post-step-route-tier-dual-write-increment.md) dual-write, pursuit increments step count, token usage, and elapsed wall-clock time in state.json so restarts resume accounting from the last good snapshot.
4. Plane D scheduling reads the same budget fields so self-improvement platform turns stay bounded and do not displace the active project goal during [A3.2](A3.2-goal-autopilot-until-goal-verify-or-hard-block.md) goal autopilot.
5. When caps are exhausted and human intervention can extend them, pursuit surfaces H2 with the structured stop reason; if budget fields are missing, contradictory, or counters drift from journal evidence, preflight fails closed at H2 rather than continuing unbounded pursuit.

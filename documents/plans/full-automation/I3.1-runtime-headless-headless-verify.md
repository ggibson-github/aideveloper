## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. Scheduled CI or an operator invokes `scripts/headless-verify.py`, which reads `state.json`, task cards, and configured evidence paths from disk without spawning a conductor LLM turn unless explicitly configured.
2. The script runs [B1.1](B1.1-s0-deterministic-mandatory-first.md) verify-router and validate-workflow for the active pursuit step, executing the same Test commands task cards declare for IDE verifier workers.
3. Passing runs append immutable logs under `evidence/` per the [H4](H4-evidence.md) contract so headless output aggregates into goal_verify the same way IDE-captured evidence does.
4. On failure, headless-verify sets `last_verify` to failed in `state.json` and records log paths so the conductor can resume on the next IDE continue without assuming prior checks still hold.
5. If `state.json` is corrupt, required task cards are missing, or evidence paths are stale, headless mode fails closed and surfaces structured H2 rather than marking verification passed.

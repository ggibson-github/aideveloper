## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. After design artifacts change (HLD, DD, diagrams, task cards), S0 update-staleness.py derives dependency edges and sets stale flags in docs/manifest/staleness.json without conductor improvisation.
2. On each pursuit wake before implement spawn, preflight reads staleness.json; paths touching stale design nodes block worker spawn until reconcile-stale clears flags or records an H2 waiver.
3. In program mode, artifact graphs from [C4.4](C4.4-artifact-graph-per-program-and-pack.md) complement the design manifest so cross-stream contracts surface stale nodes the global manifest alone might miss.
4. When vision or hierarchy source changes, affected node ids enqueue deepen items so the staleness graph and pursuit queue stay aligned with approved scope.
5. If staleness.json is missing, corrupt, or shows unresolved stale nodes on a path about to implement, pursuit stops at H2—silent implement against obsolete design violates Plane E and undermines goal_verify evidence.

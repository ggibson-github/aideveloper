## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. Each product turn executes exactly one `next_action` from state.json—one task card or one skill phase aligned with Plane C pipeline order—without skipping to push or refactor while implement evidence is missing.
2. [B1.1](B1.1-s0-deterministic-mandatory-first.md) sync-state and route-tier read state.json first when available; the conductor interprets the journal mirror only when S0 output is silent.
3. Under interleaved [B4.2](B4.2-platform-promotion-queue-peek-drain.md) scheduling, product turns still advance `next_action` atomically so Plane C progress does not desync from recorded state.
4. Workers execute the delegated slice of `next_action`; only the conductor advances past the task after verify passes or structured H2 is recorded per [G1.3](G1.3-last-verify-before-advance.md).
5. If journal and state.json disagree on `next_action`, or required evidence is missing, [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) blocks the turn until dual-write reconciles.

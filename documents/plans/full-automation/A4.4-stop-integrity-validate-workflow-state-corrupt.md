## Reader narrative
<!-- prose-source: agent plane-a 2026-06-28 -->

Integrity stops fire when the harness cannot trust its own state: `validate-workflow.py` fails, state.json is corrupt or schema-invalid, dual-write mismatch between journal and state, or artifact graph references missing nodes. Pursuit must not continue on ambiguous memory.

Recovery is deterministic-first: repair scripts, restore snapshot from Plane H, or escalate H2 with the validation log attached. Continuing pursuit on corrupt state duplicates work or attaches evidence to the wrong goal. This stop class is rare in production but critical for restart safety after manual edits or merge conflicts.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When preflight or post-step S0 runs `validate-workflow.py` against journal and state.json, any schema failure, conformance error, or dual-write mismatch triggers an A4.4 integrity stop before another [A2.2](A2.2-if-ready-execute-one-pipeline-step.md) phase is scheduled.
2. Pursuit halts immediately—the conductor does not spawn workers or advance next_action while harness memory is untrusted, so restart after manual edits or merge conflicts cannot duplicate or skip work on ambiguous state.
3. Recovery is deterministic-first: run repair scripts, restore a Plane H snapshot, or reconcile journal and state.json until validate-workflow passes and [A2.3](A2.3-post-step-route-tier-dual-write-increment.md) dual-write restores a single source of truth.
4. If automated repair cannot restore valid state, preflight escalates H2 with the validation log attached and goal.state stays blocked until an operator approves recovery—never continuing pursuit on corrupt memory.
5. On the next wake after recovery, [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) re-runs integrity checks fail-closed so evidence paths and pursuit counters cannot attach to the wrong goal_id.

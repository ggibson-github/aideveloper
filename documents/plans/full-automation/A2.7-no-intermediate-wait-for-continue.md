# A2.7: no intermediate wait for continue

**Parent:** [A2-index](A2-index.md) · **Branch A** · **Vision §3** · **Release:** v2.15

## Reader narrative
<!-- prose-source: agent pilot 2026-06-28 -->

Earlier automation models asked humans to type “continue” after every step. This capability forbids intermediate wait-for-continue: the loop runs autonomously between H1, H2, and H3 touchpoints.

“Continue” in operator language now means one pipeline step when manually invoked—not a prompt between every phase inside autopilot. That redefinition is what makes 100% automation between touchpoints achievable.

Confusion here causes products to re-introduce manual babysitting. This spec is the guardrail against regressing to burst-and-wait workflows.

## Purpose

A2.7 defines no intermediate wait for continue for the agent-driven expert system. Pursuit & control plane — autonomous loops until goal verified.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. With autopilot.active true, the conductor runs the [A2.6](A2.6-loop-until-blocked-budget-achieved-h3-reject.md) canonical loop—[A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) preflight, one [A2.2](A2.2-if-ready-execute-one-pipeline-step.md) phase, [A2.3](A2.3-post-step-route-tier-dual-write-increment.md) post-step—back-to-back in the same session without prompting the operator to type "continue" between skill phases.
2. Manual `/continue` or operator "continue" executes exactly one pipeline step only when autopilot is inactive; it does not babysit each phase inside an active pursuit loop.
3. Permitted human pauses are H1 plan approval, H2 blocker resolution, and H3 final sign-off per [INTRO-1.2](INTRO-1.2-human-touchpoint-contract-h1-h2-h3.md); check-pipeline-blocked BLOCKED exits the loop until H2 clears.
4. Skills, rules, and run-local-pipeline must not reintroduce intermediate wait-for-user gates between post-step dual-write and the next preflight except at H1, H2, or H3 touchpoints.
5. If a conductor turn ends while autopilot remains active without advancing pursuit or dual-writing an H2 stop reason, the next wake fails closed at preflight BLOCKED until journal and state.json reconcile the stall.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. Before [I2.1](I2.1-runtime-sdk-run-local-pipeline-goal-autopilot.md) invokes Cursor CLI or cursor-sdk for a headless pursuit turn, preflight loads `CURSOR_API_KEY` from environment or the operator secrets store listed in docs/facts/INDEX—never from committed repo files.
2. Optional `AUTOPILOT_MODEL` overrides the default slug for autopilot batches, but [J1](J1-model-policy.md) and model-policy.json still bind conductor roles to genius tier and worker roles to economy—override cannot bypass subagent_models routing.
3. Each SDK daemon wake passes the resolved API key and model selection into Agent.prompt or headless CLI (`agent -p`); authentication failures abort the turn immediately with structured H2 citing credential setup steps.
4. Credential rotation mid-pursuit requires the operator to update environment or the secrets store; the next daemon wake reloads config, dual-writes the change to journal and state.json, and resumes only after preflight re-authenticates.
5. Missing or invalid credentials never fall back to unauthenticated API calls—pursuit stops fail-closed with H2 and may notify via [I5.2](I5.2-runtime-notify-webhook-email-h2-h3-only.md) rather than silently retrying.

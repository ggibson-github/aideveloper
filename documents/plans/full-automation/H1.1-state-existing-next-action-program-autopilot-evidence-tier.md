## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. On every pursuit wake, S0 reads top-level `next_action`, `program`, `autopilot`, `evidence_required`, `model_tier`, and `spawn_workers` from `journal/state.json` before the conductor interprets `journal/progress.md` ([B1.1](B1.1-s0-deterministic-mandatory-first.md)).
2. `route-tier.py --apply` and `validate-workflow.py` consume these legacy router fields to select pipeline phase, model tier, and worker spawn policy without re-deriving state in the LLM.
3. Additive v2 blocks (`goal`, `platform`, `pursuit`, `hitl`, `company`) extend the router atomically via journal-keeper and `sync-state.py` while preserving existing consumer keys per [APP-B](APP-B-state-json-sketch.md).
4. When `autopilot.active` is true, `next_action` advances only through S0 apply paths—manual journal edits without dual-write do not change the machine router.
5. Corrupt or partial `state.json` triggers [A4.4](A4.4-stop-integrity-validate-workflow-state-corrupt.md) integrity stop with H2; pursuit does not resume until validate-workflow passes.

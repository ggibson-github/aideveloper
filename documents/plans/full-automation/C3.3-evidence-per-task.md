## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. During implement or verify turns, economy workers detect repeated file patterns, test fixes, or catalog misses across pursuit iterations and emit a promotion flag in their return contract—the conductor receives the signal; workers never mutate `platform.promotion_queue` directly.
2. The genius-tier conductor merges worker flags after each pursuit step, deduplicates by content fingerprint, and enqueues one `promotion_queue` item per confirmed pattern at the appropriate `target_level` from the [D1](D1-index.md) promotion ladder.
3. Structural rework invisible to shell history—duplicate scaffolding, recurring verify failures on the same fixture, or compose-first misses—routes here; pair flags with [B4.4](B4.4-divergence-log-when-not-composing.md) divergence logs when catalog composition failed before invention.
4. A single worker flag defers enqueue until a second independent flag or the [D2.1.1](D2.1.1-enqueue-repeated-manual-command-2x.md) manual 2× command trigger confirms the pattern, avoiding false-positive platform work.
5. If workers write queue JSON directly, deduplication fails and duplicate promotion items stall the parallel queue, pursuit fails closed at H2 until the conductor reconciles flags and dual-writes a single enqueue record per pattern.

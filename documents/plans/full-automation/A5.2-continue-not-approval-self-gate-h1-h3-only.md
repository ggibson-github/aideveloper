**Parent:** [A5-index](A5-index.md) · **Branch A** · **Vision §3** · **Release:** v2.15

## Reader narrative
<!-- prose-source: agent plane-a 2026-06-28 -->

Continue is not approval. Typing Continue or running `/continue` does not waive HLD, DD, feature design, or manifest gates—those become self-gate plus evidence where automatable, with H1 and H3 remaining the only default human approvals unless strict HITL is enabled.

Misreading Continue as “yes, proceed anyway” reintroduces silent gate skips. The conductor and S0 scripts treat unresolved blocking questions and pending human gates as BLOCKED preflight outcomes. Only explicit approval phrases or recorded waivers in the journal clear design gates.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When the operator types Continue or `/continue`, [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) checks gates_pending, blocking_questions, and unresolved HLD/DD/feature-design waits—treating them as BLOCKED per [A4.1](A4.1-stop-human-h1-h2-h3.md) even if other fields look READY.
2. Continue does not waive HLD, DD, manifest, or feature-design gates; automatable design checks run as self-gate plus evidence, leaving H1 plan approval and H3 sign-off as the default human touchpoints per [INTRO-1.2](INTRO-1.2-human-touchpoint-contract-h1-h2-h3.md).
3. Only explicit approval phrases (e.g. "I approve the HLD") or recorded waivers in journal Resolved Q&A clear design gates—[A5.1](A5.1-continue-resume-pursuit-if-not-blocked.md) never interprets silence or repeated Continue as consent.
4. S0 scripts in route-tier and validate-workflow enforce the same rule headlessly so [A3.4](A3.4-sdk-daemon-run-local-pipeline-24-7.md) daemons cannot skip gates because no human watches the chat.
5. If an operator believed Continue cleared a gate but preflight still blocks, pursuit dual-writes H2 with the pending gate id and re-prompts for substantive approval—not another pipeline phase.

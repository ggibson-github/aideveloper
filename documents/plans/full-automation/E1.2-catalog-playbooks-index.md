# E1.2: catalog playbooks INDEX

**Parent:** [E1-index](E1-index.md) · **Branch E:** Knowledge & composition — catalog, compose-first, staleness. · **Vision §7**

## Reader narrative
<!-- prose-source: agent plane-e 2026-06-28 -->

`docs/playbooks/INDEX.md` catalogs repo-scoped playbooks distilled after features complete—patterns for implement, verify, git-workflow, and pack-specific flows. Playbooks sit between skills (procedure) and facts (external truth) in the reuse ladder.

The INDEX is a first-class catalog node referenced by [E2.3](E2.3-compose-rank-script-playbook-skill-facts.md) ranking (script beats playbook beats skill) and by playbook-keeper after delivery. Compose-first ([B4.3](B4.3-compose-first-catalog-before-improvise.md)) requires playbook search before inventing new procedural guidance. Staleness marks playbook nodes when upstream design changes per [E5.1](E5.1-staleness-design-graph-staleness-json.md).

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. Before procedural work that lacks an S0 script, the conductor or librarian queries `docs/playbooks/INDEX.md` per [B4.3](B4.3-compose-first-catalog-before-improvise.md) compose-first—prefer an indexed playbook over improvising new guidance.
2. [E2.3](E2.3-compose-rank-script-playbook-skill-facts.md) ranks playbooks above skills; when a ranked playbook matches, the conductor loads its path from INDEX and sets economy workers' allowed_reads to that slug only.
3. After feature delivery, [D4.1](D4.1-platform-work-playbook-keeper.md) distills repeated pursuit patterns into `docs/playbooks/<slug>.md` and registers the slug in INDEX so the next goal inherits L1 reuse.
4. When upstream design changes, [E5.1](E5.1-staleness-design-graph-staleness-json.md) marks playbook catalog nodes stale; reconcile-stale blocks compose until INDEX rows match current vision and playbook bodies.
5. If INDEX lookup fails, the playbook path is missing, or compose proceeds without catalog search, pursuit stops at H2 with structured dual-write—never treat improvised procedure as cataloged reuse.

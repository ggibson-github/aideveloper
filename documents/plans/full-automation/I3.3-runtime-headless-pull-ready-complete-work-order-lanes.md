## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. At H1 plan approval, the conductor stamps goal_id, optional parent_goal, and goal_type into state.json and journal—establishing the identity record every subsequent wake reloads before routing.
2. On each pursuit wake, preflight reads goal_type to select contracts: milestone goals require manifest gates, feature goals bind task cards and evidence paths, and company_ops goals follow the ops queue.
3. When parent_goal is set, child pursuits inherit roll-up scope so parallel program streams report status to the parent without duplicating identity fields at each leaf.
4. The conductor and journal-keeper dual-write any H1 amendments to these fields; S0 scripts refuse progression if goal_id duplicates or parent_goal points to a missing ancestor.
5. If identity drifts or conflicts—duplicate ids, wrong parent—pursuit halts at H2; evidence must not attach to the wrong scope, and goal verification cannot pass until fields are reconciled.

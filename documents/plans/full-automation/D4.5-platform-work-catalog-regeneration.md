## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. After new playbooks, scripts, skills, or pack fragments land, platform work runs S0 catalog regeneration when scripts exist; otherwise an economy worker with `allowed_reads` limited to new artifact paths only.
2. The worker updates `docs/playbooks/INDEX.md`, manifest catalog nodes, and operator discovery rows so [B4.3](B4.3-compose-first-catalog-before-improvise.md) compose-first search finds fresh artifacts.
3. Every INDEX row must resolve to a real file on disk—regeneration without artifact creation is invalid and fails [D6.1](D6.1-platform-done-catalog-index-row.md).
4. On verify pass, the conductor dequeues the promotion item; librarian and implement workers on subsequent product turns see updated catalog paths.
5. If INDEX rows are stale, broken, or point at missing files, pursuit blocks downstream compose-first routing at H2 until regeneration completes.

# E3.1: facts docs/facts external truth

**Parent:** [E3-index](E3-index.md) · **Branch E** · **Vision §7** · **Release:** v2.17

## Reader narrative
<!-- prose-source: agent plane-e 2026-06-28 -->

`docs/facts/` holds external truth separate from design and journal narrative: environment URLs, schema snippets, operator-provided credentials pointers, team routing. Agents read facts; they do not invent URLs or API shapes when INDEX lists a topic file.

Facts integrate with compose at lowest rank ([E2.3](E2.3-compose-rank-script-playbook-skill-facts.md)) and with context retrieval ([E1.4](E1.4-catalog-facts-index.md)). Blocking questions about external config resolve by reading or extending facts—not by guessing. Stale facts when upstream systems change should trigger journal note and optional reconcile-stale ([E5.3](E5.3-staleness-reconcile-stale-artifact-graph-skills.md)).

## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. Before spawning implement workers, [Librarian](B2.2-librarian-allowed-reads-catalog-composition.md) returns `allowed_reads` (max five paths) from catalog composition; spawn contracts inject those paths into worker context so pursuit does not improvise scope.
2. implement-feature workers touch only files listed on the active task card plus `allowed_reads`—opening full design trees or drive-by refactors outside that boundary is scope creep.
3. On merge, the conductor compares the diff to task card file targets and rejects out-of-scope changes for rework rather than silent acceptance into the goal slice.
4. A first violation triggers rework with diff evidence attached under `evidence/`; repeated creep on the same task escalates to H2 with structured failure notes per this mistake-class control.
5. If a worker cannot finish within the card boundary, pursuit fails closed at H2 instead of expanding scope without conductor and operator visibility.

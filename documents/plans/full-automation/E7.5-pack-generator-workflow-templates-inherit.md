# E7.5: pack generator workflow templates inherit

**Parent:** [E7-index](E7-index.md) · **Branch E** · **Vision §19** · **Release:** v2.27

## Reader narrative
<!-- prose-source: agent transistor-expansion 2026-06-28 -->

Template-packs ship more than roles and prose playbooks: they carry **generator workflow templates** and **domain transistor libraries** that a new company inherits on day one. Under [F1.9](F1.9-pack-transistors-and-generator-workflows.md), each pack may include `workflows/*.json` seed DAGs and `transistors/*.json` blocks for its industry—game studio mesh/rig/import chains, data platform ingest/model/deploy paths, or software delivery scaffold sequences. [E7.1](E7.1-resolve-deliverable-type-workflow-template.md) resolves the goal deliverable type to one of these templates before the composer customizes nodes.

Instantiation copies template references into pursuit state; program-scoper does not silently fork pack files into the consumer repo ([F5.3](F5.3-no-repo-outside-template-packs-ceiling.md)). Cross-pack imports via [F5.1](F5.1-cross-pack-imports-micro-packs.md) may add supplemental workflow fragments. Generic blocks live in [F5.4](F5.4-shared-transistors-library-template-packs--shared.md) `_shared` and merge with pack overlay at query time. Pack CI validates all templates against workflow-dag and transistor v1 schemas before publish.

See [Vision §19 — Transistor & generator workflow model](../../full-automation-vision-and-hierarchy.md#19-transistor--generator-workflow-model) and [SEC-15-v2.27](SEC-15-v2.27-release-v2-27-pack-workflow-templates-cross-domain.md) for the release that operationalizes pack inheritance.

## Edge cases

- Operator closes laptop mid-loop — state.json must resume from last good dual-write including active_workflow.
- Transistor version bump mid-pursuit — E5.4 marks workflow stale; re-validate before next node.
- L0 waiver node without promotion progress — D3.3 priority boost then H2 if threshold exceeded.
- Pack overlay id collision — F5.4 semver fork per D5.3, not silent overwrite.
- Parallel branch join missing typed input — validate-workflow-dag fails at compose time.

## Failure modes

- **Fuzzy chain:** Implement without workflow_node_id when C6.1 applies → G5.8 blocks at preflight.
- **False complete:** Node marked done without transistor verify evidence → G2.5 goal_verify fails closed.
- **Stale workflow:** active_workflow.validation_hash mismatch → E5.4 reconcile before advance.
- **Duplicate transistor:** G5.6 list-transistors --check-duplicates rejects promotion.
- **Scope bleed:** Worker runs transistors outside bound node → C6.3 conformance failure.

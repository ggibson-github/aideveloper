## Reader narrative

Local subagents run as economy-tier Task/explore/shell workers under conductor contracts with capped allowed_reads. The IDE runtime spawns them via orchestrate-subagents; workers return summaries without touching journal or state.json.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When spawn_workers is true and the conductor needs bounded context, orchestrate-subagents spawns economy-tier Task, explore, or shell workers with contracts that cap allowed_reads and forbid journal or state.json writes.
2. Before each spawn, route-tier.py reads state.json and sets model_tier so workers run at economy tier—not ad-hoc model picks per invocation.
3. Parallel local subagents start only when the integration manifest lists independent lanes per [C4.2](C4.2-orchestrate-program.md); otherwise the conductor serializes worker turns.
4. Workers return summaries to the parent; the conductor alone merges results and dual-writes journal and state.json after the pipeline step completes.
5. Failed or timed-out worker runs append to [H5](H5-worker-runs.md); persistent failures surface as H2 blockers without marking the pursuit step complete.

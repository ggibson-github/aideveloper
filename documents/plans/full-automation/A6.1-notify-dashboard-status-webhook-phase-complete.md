**Parent:** [A6-index](A6-index.md) · **Branch A** · **Vision §3** · **Release:** v2.23

## Reader narrative
<!-- prose-source: agent plane-a 2026-06-28 -->

Notifications inform; they do not block pursuit. Phase completion—design artifact published, implement task verified, goal transition—updates `docs/operator/dashboard.md`, `STATUS.md`, and optional webhooks so operators have situational awareness without approving each step.

Regeneration is S0 where `generate-dashboard.py` exists. Webhooks fire on meaningful milestones, not every token, to avoid alert fatigue. This capability satisfies observability for unattended runs while preserving [A6.3](A6.3-operator-observe-without-unblocking-loop.md): reading STATUS never substitutes for H2 resolution.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. After each meaningful phase completion—design artifact written, implement task verified with evidence, or goal state transition—journal-keeper invokes [S0](B1.1-s0-deterministic-mandatory-first.md) `generate-dashboard.py` to regenerate `docs/operator/dashboard.md` and `STATUS.md` without blocking the next pursuit turn.
2. Configured webhooks fire on milestones (phase complete, H3 pending, goal achieved)—not every conductor turn—so [A3.2](A3.2-goal-autopilot-until-goal-verify-or-hard-block.md) unattended runs remain observable without alert fatigue per [A6.2](A6.2-notify-digest-on-h2-blocker-not-every-step.md).
3. Notifications dual-write snapshot fields in state.json for headless consumers ([I5.1](I5.1-runtime-notify-status-dashboard-generation.md)) while pursuit continues; notification failure never blocks [A2.2](A2.2-if-ready-execute-one-pipeline-step.md) execution.
4. Reading STATUS or dashboard output does not clear H2 blockers or substitute for operator action—[A6.3](A6.3-operator-observe-without-unblocking-loop.md) preserves strict HITL separation.
5. If dashboard regeneration fails, the conductor logs the error and optionally enqueues a platform notification item—pursuit advances only when preflight is READY, not when the operator acknowledged the digest.

## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. Goal autopilot reads budget fields from A1.4—steps, tokens, wall-clock—deciding whether pursuit is unlimited or capped at daily checkpoints after H1 approval.
2. Unlimited pursuit maximizes autonomy; checkpoint mode decrements budgets each turn and fires A4.3 resource stops when exhausted, with dashboard visibility per I5.1.
3. Budget exhaustion is a hard stop in the A4 taxonomy—not a soft warning—so operators must explicitly raise caps or approve continuation at H2 before pursuit resumes.
4. Until an ADR resolves unlimited versus checkpoint default, pursuit uses conservative daily step caps outside production while production requires operator opt-in via H1 plan fields.
5. If budget counters drift from state.json, checkpoints are bypassed, or the dashboard cannot show remaining budget, pursuit fails closed at H2 rather than silently burning tokens.

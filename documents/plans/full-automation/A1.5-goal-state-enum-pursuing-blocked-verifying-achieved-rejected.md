# A1.5: goal_state enum pursuing blocked verifying achieved rejected

**Parent:** [A1-index](A1-index.md) · **Branch A** · **Vision §3** · **Release:** v2.14

## Reader narrative
<!-- prose-source: agent pilot 2026-06-28 -->

Operators and dashboards need a single, honest lifecycle state for each goal. This capability defines the goal state enum: pursuing, blocked, verifying, achieved, and rejected—transitions driven by preflight, step execution, goal_verify, and H3 outcomes.

The enum prevents ambiguous records such as “done” without verification or “in progress” while H3 is pending. Blocked correlates with H2 assistance; verifying means scope is complete and goal_verify is running; rejected captures H3 denial with notes so pursuit can resume with corrected criteria.

State must stay synchronized across journal/progress.md and state.json. Resume after laptop sleep or daemon restart relies on this enum being authoritative. See the A2 pursuit loop for when each transition fires in sequence.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. At H1 plan approval, the conductor sets goal.state to pursuing in state.json—dual-written to journal—alongside [A1.1](A1.1-goal-id-parent-goal-goal-type.md) identity fields so every wake and dashboard reads the same lifecycle starting point.
2. On each [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) wake, READY keeps goal.state at pursuing while BLOCKED sets blocked and routes to H2 assistance; preflight never schedules pipeline phases when the enum is blocked or verifying.
3. When [A2.4](A2.4-goal-scope-complete-run-goal-verify.md) marks scope complete, the conductor transitions goal.state to verifying and runs [A1.3](A1.3-goal-verify-command-meta-test.md) goal_verify before requesting operator sign-off.
4. Passing goal_verify advances to [A2.5](A2.5-goal-verify-pass-transition-h3-pending.md) H3 pending without claiming achieved; H3 acceptance sets achieved while H3 rejection records structured notes and returns goal.state to pursuing with corrected criteria.
5. If journal and state.json disagree on goal.state after laptop resume or concurrent edit, preflight fails closed at H2 until journal-keeper reconciles the enum before pursuit continues.

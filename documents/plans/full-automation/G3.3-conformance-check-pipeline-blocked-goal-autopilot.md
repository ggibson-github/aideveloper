## Behavior / step logic
<!-- timeline-source: agent cli-composer-2.5 2026-06-28 -->

1. `scripts/headless-verify.py` runs without an IDE session, reading `journal/state.json` and the active task card from disk to delegate verification to `verify-router.py` as an [S0](B1.1-s0-deterministic-mandatory-first.md) check—no conductor LLM turn unless explicitly configured.
2. Headless mode mirrors the IDE verifier contract: it executes file-derived test or tool commands from the task card and does not improvise scope beyond what state and cards declare.
3. On exit 0, evidence logs land under `evidence/` with the same immutability rules as the IDE verifier skill ([H4](H4-evidence.md)), and `last_verify` is set to passed in state.json for the conductor to read on the next wake.
4. Operators invoke headless-verify locally between IDE sessions, and CI jobs ([I3.2](I3.2-runtime-headless-github-actions-validate-verify.md)) reuse the same entry point so implement batches get verified without consuming a genius-tier orchestration turn.
5. On non-zero exit, headless-verify sets `last_verify` to failed in state.json; pursuit fail-closes per [G1.3](G1.3-last-verify-before-advance.md) until the conductor or operator reruns verify with green evidence—CI failure alone does not clear H3 or design gates.

# A1.2: success_criteria machine-checkable

**Parent:** [A1-index](A1-index.md) · **Branch A** · **Vision §3** · **Release:** v2.14

## Reader narrative
<!-- prose-source: agent pilot 2026-06-28 -->

A goal that cannot be checked mechanically is not a goal the system can pursue autonomously. This capability requires success_criteria that scripts or tests can evaluate—clear predicates, exit codes, or artifact checks—not subjective “looks good” language.

Success criteria are the bridge between intent (approved at H1) and [goal verification](A1.3-goal-verify-command-meta-test.md). Each criterion should map to a command, file presence, or metric the verifier can run without asking a human. Vague criteria force H2 blockers and stall the always-on loop.

When criteria change mid-pursuit, the conductor must dual-write the update and re-run preflight so downstream steps do not assume an obsolete definition of done. Poor criteria are a common root cause of false “achieved” states; this spec exists to prevent that class of failure.

## Purpose

A1.2 defines success criteria machine checkable for the agent-driven expert system. Pursuit & control plane — autonomous loops until goal verified.

## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. At H1 plan approval, the conductor records `success_criteria[]` in `state.goal` as machine-checkable predicates—exit codes, artifact paths, or metric thresholds—that [A1.3](A1.3-goal-verify-command-meta-test.md) and the verifier can run without human judgment.
2. On each pursuit wake, [A2.1](A2.1-preflight-check-pipeline-blocked-extended.md) rejects criteria that cannot map to a script, test, or artifact check; subjective language forces an H2 blocker before any implement step proceeds.
3. The conductor binds each criterion to a concrete verify command or evidence path in task cards so workers know which checks must pass before `goal.state` may advance past pursuing.
4. If operators amend success_criteria mid-pursuit, the conductor dual-writes journal and state.json, then re-runs preflight so downstream steps do not pursue an obsolete definition of done.
5. When verification cannot reconcile criteria against collected evidence, pursuit fails closed at H2 rather than transitioning to achieved or H3 pending.

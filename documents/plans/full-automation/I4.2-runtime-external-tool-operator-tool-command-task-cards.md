## Behavior / step logic
<!-- timeline-source: agent cursor-agent 2026-06-28 -->

1. When an implement task card declares a Tool command block, the conductor spawns a tool-operator shell worker instead of a phase implement worker, binding execution to the literal CLI on the card.
2. The tool-operator worker runs pack-declared commands—Blender batch exports, Unreal commandlets, custom CLIs—and may write outputs only to paths explicitly declared on the task card.
3. tool-operator skill bounds the command surface and filesystem writes; product source edits remain the domain of normal implement workers, not shell tool runs.
4. On completion, stdout/stderr and artifact paths attach to evidence/ and route through [B2.4](B2.4-verifier-tool-operator-evidence.md) before last_verify can pass and pursuit advances.
5. Tool failures, missing outputs, or ad-hoc invocations without a bound task card fail closed with verify escalation—pursuit turns stop rather than improvising undocumented tooling.

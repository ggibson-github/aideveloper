## Behavior / step logic

1. L2 promotion moves repeatable playbook steps into deterministic scripts under `scripts/`—routing, validation, queue peek, and verify dispatch per [D4.2](D4.2-platform-work-script-extraction.md) script extraction on platform turns.
2. At pursuit wake, [B1.1](B1.1-s0-deterministic-mandatory-first.md) mandatory-first requires the conductor to run applicable L2 scripts and use their output before any worker re-implements the same logic in chat.
3. Each promoted L2 script carries a unit test path under tests/unit/ and, when it replaces manual verify, registers with [G1.1](G1.1-task-verify-router-verifier.md) verify-router per [D6.2](D6.2-platform-done-verify-script-l2.md).
4. Platform workers complete L2 promotion only when the script exists, tests pass, and catalog or playbook references point to the script—dual-write promotion_queue completion to journal and state.json.
5. If script extraction marks done without a test, verify-router cannot dispatch the script, or workers duplicate script behavior inline, pursuit fails closed at H2 with the promotion item reopened—never false-complete L2 packaging.
